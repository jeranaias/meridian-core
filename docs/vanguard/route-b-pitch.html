The Pattern

Same code, different chip.

The autopilot stack is hardware-blind. It calls abstract trait interfaces — UartDriver, SpiDevice, I2cDevice, etc. The simulator and the chip each provide their own implementations. Switching is one line: SitlHal::new() becomes Stm32Hal::new().

Why this matters

  • Risk is bounded. The code that flew in simulator across 1,300+ scenarios is the same code that runs on the chip. We're not rewriting flight logic — only providing a new platform shim.
  • Tests transfer. Every SITL scenario keeps validating the autopilot logic during integration. Regression catches happen instantly.
  • Iteration is fast. Tweak a control loop in the laptop simulator, prove it under wind/disturbance, then flash the same change to the boat.
  • Future-proof. Same pattern lets us swap to Pixhawk 6X, Cube Black, or a custom flight controller later — single platform crate per board, autopilot logic untouched.

The Plan

Day-by-day to a flying autopilot.

Honest schedule. Day 1 is the integration glue — that's the new code. Days 2–7 are hardware bring-up: catching the surprises that the simulator can't see. Day 8 onwards is mission rehearsal.

Day 1
Integration glue
Write main.rs wiring EKF, modes, mission, failsafe, drivers into the H7 binary. ~300 lines, follows the existing simulator main.rs pattern.
Day 2
Compile + first boot
Linker fixes, missing trait impls, sensor pin assignments for the Cube. First boot on real chip. Establish the iteration loop.
Day 3
USB CDC continued
Continue from tonight's session. Without USB, every iteration costs a bench-power cycle. With it, fully remote.
Day 4–5
Sensor bring-up
IMU streaming at 1 kHz. Baro at 50 Hz. Compass calibrating. Each driver is unit-tested already, but real hardware always has surprises (clock skew, bus contention).
Day 6
Orca live · GPS healthy
Wire Orca's NMEA 0183 output to a Cube UART. EKF gets first GPS-fused state. Watch position lock on the dock.
Day 7
PWM out · rudder calibration
Drive the rudder from MANUAL mode via the tablet. Verify direction and travel. This is the first time the autopilot moves a real surface.
Day 8–10
Modes · failsafe · mission
MANUAL, HOLD, AUTO, RTL working end-to-end. Failsafe verified by yanking the GPS feed mid-mission. Mission engine drives a rectangular waypoint loop on the bench.
Day 11+
Wet test
Boat in the water at the slip. Tablet connected over the radio. First on-water mission.
